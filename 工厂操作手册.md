# MACA 离线工厂操作手册

> 如何启动整个"规律发现"流程，以及如何指导它生成你想要的案例。
> 配套阅读：`项目说明与上手指南.md`（架构与模块细节）。

本手册只讲一件事：**怎么让离线工厂跑起来、怎么把它引导到你关心的场景**。

---

## 一、一分钟跑通（最小步骤）

```bash
cd Code
conda activate llm_control            # 或任何装了依赖的环境

# 1) 先零成本试跑（无需 API key，用 Mock 智能体）
python -m maca.factory --batch 1 --mock

# 2) 配好真实 key 后，跑一个真实批次
#    在 Code/.env 里写：OPENAI_API_KEY=sk-...
python -m maca.factory --batch 2 --source generate

# 3) 边跑边看工厂在干什么（另开一个终端）
tail -f Results/factory/library_growth.log
```

跑完后库文件（`maca/library/rules.jsonl` / `cases.jsonl`）会长大，
`python -m maca.run --stats` 能看到规则/案例数量变化。

---

## 二、整个流程是怎么运转的（一张图）

```
你启动工厂批次
   │
   ├─【阶段1】消化运行时反馈队列 queue.jsonl（如果有）
   │      在线跑场景时"没覆盖/有冲突"的场景会自动进这个队列
   │      → 委员会深思 → 变成案例
   │
   └─【阶段2】规律发现（每个 --batch 计数生成 1 个场景族）
          生成智能体(gpt-5.5) 设计一个"参数化场景族"
             │  例：左侧横穿卡车，扫 ego_speed × ttc × lateral_speed
             ▼
          确定性实例化 + 网格扫描（对每个参数点仿真全部 8 种机动）
             │
             ├─ 规律格点（某机动明显最优）→ 归纳成规则 → 数值复核 → 晋升
             └─ 模糊格点（多个机动差不多/涉及行人）→ 委员会深思 → 变成案例
          批末：抽样复验现役规则（防漂移）
```

**关键点**：LLM 只在离线这里花钱；每次调用前都会检查"库是不是已经覆盖了"，
已覆盖的直接跳过不花 token。所以反复跑同一区域不会浪费。

## 三、三种启动方式（按你的目的选）

### 目的 A：让工厂自主探索、自动补库（最省心）

```bash
python -m maca.factory --batch 3
```

生成器会读"覆盖简报"（当前库覆盖了哪些区域），自己去找**没覆盖的空白区**
或**现役规则的边界**来设计实验。`--batch N` = 生成 N 个场景族。
适合：你不确定缺什么，让系统自己查漏补缺。

### 目的 B：只把在线遇到的"疑难场景"补上（针对性最强）

```bash
# 先在线跑一些场景，未命中的会自动入队
python -m maca.run --scenario <你的场景>
# 再让工厂只消化队列，不生成新场景
python -m maca.factory --batch 0 --source queue
```

适合：你有具体场景库处理不好，想精确补齐。`--batch 0` 表示不生成新族。

### 目的 C：零成本演示/回归（无需 API key）

```bash
python -m maca.factory --batch 3 --mock
```

用确定性 Mock 智能体（轮换 3 个预置场景族），跑通完整流程。
适合：验证流水线、演示、CI 回归。

## 四、如何指导它生成"我想要的案例"

工厂的场景由**生成智能体**设计，你有四个层次的抓手，从软到硬：

### 抓手 1：调生成器的"设计原则"提示词（影响它的整体偏好）

文件：`maca/factory/generator.py` 里的 `_SYSTEM`（"Design principles" 段）。
例如想让它多探索中等 TTC（避撞仍有区分度的区间）而不是极短 TTC，
就在设计原则里写明。**已按你的要求把 TTC 引导到 0.6–1.6 s 的决策丰富区间**
（极短 TTC 下所有机动都撞，学不到有区分度的规律）。

### 抓手 2：改参数护栏范围（硬限制生成器能选的区间）

文件：`maca/factory/generator.py` 顶部的 `_SANE_RANGES`：

```python
_SANE_RANGES = {
    "ego_speed": (5.0, 30.0),     # 自车速度 m/s
    "ttc": (0.4, 2.4),            # 碰撞时间 s ← 想限制 TTC 范围改这里
    "lateral_speed": (2.0, 12.0), # 横向逼近速度 m/s
    "gap": (3.0, 60.0),           # 纵向间距 m
    "speed_diff": (0.0, 12.0),    # 与前车速差 m/s
    "lateral_offset": (3.0, 6.0), # 切入横向偏移 m
}
```

生成器选的任何范围都会被裁进这里。**想强制只在某区间实验，收窄对应上下界即可。**

### 抓手 3：指定场景模式（pattern）

生成器可选 4 种模式，想偏向某种可在设计原则里点名，或直接看哪些模式
还没有规则覆盖（`--stats` + 看 rules.jsonl 的 provenance）：

| 模式 | 含义 | 需要的参数 |
|---|---|---|
| `lateral_crossing` | 目标从侧向横穿（卡车横穿路口等） | ego_speed, ttc, lateral_speed |
| `lead_braking` | 前车急刹 | ego_speed, gap, speed_diff |
| `cut_in` | 邻道车辆切入 | ego_speed, gap, speed_diff |
| `static_blockage` | 静态障碍堵路 | ego_speed, gap |

模式的上下文（道路拓扑、行人有无及位置、道路边界、是否有跟车、
哪侧走廊被堵）也由生成器在模板里设定，可在设计原则里引导。

### 抓手 4：手工放一个种子场景，让工厂围绕它泛化（最精确）

```bash
# 1) 在 maca/scenario/scenarios/ 写一个 <name>.json（格式见上手指南 §13.1）
# 2) 在线跑它，未命中则入队
python -m maca.run --scenario <name>
# 3) 工厂消化队列时会围绕这个场景的特征生成案例
python -m maca.factory --batch 0 --source queue
```

适合：你有一个非常具体的真实场景，想让库精确覆盖它及其邻域。

## 五、跑完怎么检查成果

```bash
# 库规模变化
python -m maca.run --stats

# 逐事件看工厂做了什么（生成了哪些案例、判断、完善了哪些规则）
tail -50 Results/factory/library_growth.log

# 看某批次的完整机器可读报告
ls -t Results/factory/batch-*.json | head -1 | xargs cat

# 全场景在线回归（确认扩库没破坏已有决策）
for s in $(python -m maca.run --list); do
  python -m maca.run --scenario $s 2>/dev/null | grep "^\[Decision\]"
done
```

`library_growth.log` 里每条新案例都有三行：**决策机动码 → 仿真判断（结局
严重度）→ 评估（是否有效 + 可迁移教训）**，这是审阅工厂产出质量的主要窗口。

## 六、常见调整速查

| 我想… | 怎么做 |
|---|---|
| TTC 别老是极短，覆盖更真实区间 | 已在 `_SYSTEM` 引导到 0.6–1.6 s；如需更宽改 `_SANE_RANGES["ttc"]` |
| 只探索某种场景（如前车急刹） | 在 `_SYSTEM` 设计原则点名 pattern，或收窄其它模式的参数 |
| 多生成规则、少生成案例 | 引导生成器去"规律明显"的区间（不同机动严重度差距大） |
| 多生成案例（模糊权衡） | 引导到"多个机动严重度接近"或"涉及行人权衡"的区间 |
| 换更强/更省的生成模型 | `MACA_MODEL_GENERATOR=gpt-5.5`、`MACA_EFFORT_GENERATOR=high` |
| 一次生成更多场景族 | `--batch N` 调大 N（真实模式每族约 1–3 分钟） |
| 完全离线试跑 | 加 `--mock` |

## 七、注意事项

- **成本**：真实批次每个场景族要多次调用 gpt-5.5（生成+归纳+若干委员会深思），
  约 1–3 分钟/族。先用 `--mock` 验证流程，再花真实预算。
- **幂等安全**：反复跑不会重复劳动 —— 覆盖门控会跳过已覆盖区域。
- **规则会自动纠错**：归纳出的规则要过网格数值复核和晋升电池；每批次还会
  抽样复验现役规则，失效的自动废弃。你不需要手动清理坏规则。
- **API key**：只放在 `Code/.env`，仅离线工厂需要；在线运行完全不需要。
